# Quick Start Example

A step-by-step walkthrough of setting up and using the AI Digital Self System for the first time.

---

## Step 1: Create the directory structure

Copy the folder structure to your local machine. You only need the empty directories — content comes later.

```
your-project/
├── 00_System_Overview/
├── 01_System_Kernel/
│   └── history/
├── 02_Knowledge_Creation/
│   ├── skills/
│   ├── writing/
│   └── knowledge_notes/
├── 03_Material_Record/
│   └── history/
└── 04_Evolution/
    └── history/
```

**Time**: 5 minutes.

---

## Step 2: Write your first kernel file — Meta_Cognition

Start with the rules for how you think. This is the most important file.

Open `01_System_Kernel/Meta_Cognition.md` and write 3-5 rules:

```markdown
# Meta-Cognition

## Thinking procedure

### 1. Facts first
Before making a judgment, verify the facts.

### 2. Understand the context
Ask: what conditions produced this situation?

### 3. Long-term over short-term
When immediate feelings conflict with long-term goals, prioritize the long-term.
```

**Tip**: Start small. You can always add more rules later.

**Time**: 30 minutes to write your first draft.

---

## Step 3: Write one personality file — Value_Priorities

Pick the easiest one: **Value_Priorities** (what matters more to you).

Write 3-5 core value statements:

```markdown
# Value Priorities

### 1. Depth over breadth
I prefer deep understanding of fewer things over shallow knowledge of many.

### 2. Creation over consumption
Making something is more valuable than consuming something.

### 3. Long-term over short-term
I optimize for years, not days.
```

**Time**: 30 minutes per file. Do one per day.

---

## Step 4: Start recording material

Once a week, open `03_Material_Record/2026_05.md` and write a few lines:

```markdown
# 2026-05

## Life log
- Worked on project X
- Read two papers on topic Y

## Personality signals
- Noticed I get impatient when...
- Realized I value Z more than I thought

## External knowledge
- [Interesting article](https://example.com) — key takeaway: ...
```

**Time**: 10 minutes per week.

---

## Step 5: Run your first monthly review

After one month of recording, run the monthly trend prompt (from `Prompt_Collection.md`) with an AI assistant.

The prompt will analyze your material and produce a trend report.

**Time**: 15 minutes.

---

## Step 6: Iterate

After your first quarterly review, revise your kernel files based on what you've learned.

That's the cycle:
1. Record → 2. Review → 3. Revise → repeat

## Common pitfalls

| Pitfall | How to avoid |
|---------|-------------|
| Writing too much too fast | Start with 3-5 rules per file |
| Perfecting instead of recording | Record first, improve later |
| Editing kernel based on one event | Wait for patterns (3+ occurrences) |
| Making it public with private data | Use the `examples/` directory for public versions |
