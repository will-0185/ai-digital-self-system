# Privacy Boundary

This repository is public. It should only contain public-safe framework files, templates, skills, and examples.

## Content to keep in your private system

- Real monthly summaries
- Life logs
- Personal relationships
- Evidence indexes
- Unpublished works
- Business plans
- Sensitive personal materials
- Accounts, tokens, API keys, private keys

## Content that can be public

- System architecture
- Public templates
- Public-safe prompts
- Abstract meta-cognition rules
- Fictional examples
- Sanitized public version of system kernel after careful review

## Principle

Do not publish your full private digital self directly.
