# AI 数字分身系统

**文件定位**：本文件用于说明 AI 数字分身系统的核心目标、总体架构、文件分层、调用方式、素材记录方式、输出应用机制和演化迭代机制。

## 目标

- **核心目标**：通过长期记录和迭代构建一个越来越接近用户自身的 AI 数字分身系统。
- **系统目标**：AI 数字分身系统能够持续理解用户、辅助用户判断、帮助用户创作、延续用户的表达风格，使其在未来更强大的 AI 能力中被持续调用、持续修正、持续延展。
- **远期目标**：在更长的时间尺度上，这个数字分身能够基于用户长期积累的核心人格与表达能力继续承载用户的部分思考方式与创作能力，形成一种长期的精神延续与数字化存在。

## 核心组成

- **元认知**：思考、调用、仲裁、修正的程序。这是分身"像用户"而非"引用用户"的关键。
- **核心人格**：数字分身的内容主体。包括六个维度：世界认知、价值排序、人生方向、生存哲学、现实决策、兴趣审美。
- **元知识**：从外部资料中压缩出的少数生成性范式，数量极少但生成性极强。这是分身在专业领域推导和表达的底层基础。
- **知识库**：外部资料档案、skills、笔记等。这是分身在创作中可调用的事实和方法基础。

**四者关系**：
- 元认知是操作系统，核心人格是内容主体，元知识是底层范式，知识库是事实和方法基础。
- 元认知、核心人格、元知识共同构成"我是谁"——它们在系统内核层。知识库构成"我能调用什么"——它在知识创作层。

## 架构

```text
00_System_Overview      系统说明与配套提示词
01_System_Kernel        元认知、元知识、核心人格
02_Knowledge_Creation   外部资料档案、skills、文章、知识笔记
03_Material_Record      月度原始素材记录
04_Evolution            月度趋势、季度审查、年度演化
```

**结构说明**：
- **系统说明**(00)：方案文档和配套提示词，任何 LLM 启动时的背景资料。
- **系统内核**(01)：整个分身的核心，元认知、元知识、核心人格平行存在。
- **知识创作**(02)：创作工作区，既包含输入（外部资料档案、skills、笔记）也包含输出（文章）。
- **素材记录**(03)：底层原料，日常记录主题分块。
- **演化迭代**(04)：横向治理工具，周期性审视系统是否需要修订。

## 系统内核详解

系统内核是整个分身的"我"，八个核心文件平铺：

- **元认知**：思考的程序、调用的规则、冲突的仲裁、自我的修正
- **元知识**：从外部资料压缩的少数生成性范式
- **世界认知**：用户认为世界是什么样的
- **价值排序**：对用户来说什么更重要
- **人生方向**：用户想成为什么样的人、人生投向哪里
- **生存哲学**：用户应该怎样活
- **现实决策**：用户在具体事务上如何判断、选择和行动
- **兴趣审美**：用户被什么吸引、喜欢什么样的表达

**三类文件的职责区分**：
- **元认知**(过程)：规定怎么思考、怎么调用其他文件、怎么处理冲突、怎么修正。不产生具体内容判断。
- **元知识**(范式)：提供少数跨领域底层认知范式，每条元知识是高度抽象的判断，作为推导和表达的基础。
- **核心人格**(内容)：提供具体的世界观、价值观、人生观、决策原则、审美偏好。各自独立维护，允许保留张力。

## 运行方式

本系统采用"本地文件系统 + 双工作区"方式运行：

```
1. 系统进化工作区  → 维护并持续优化系统说明、系统内核、知识库、演化迭代
2. 内容产出工作区  → 基于当前系统完成具体内容输出
```

两个工作区的关系是：工作区1决定"我是谁"，工作区2基于"我是谁"完成"我要说什么"。工作区2的产出归档后反哺工作区1的素材记录，成为下一轮系统进化的燃料。

两者通过**本地文件系统**作为媒介进行流转，不直接互通。

### 本地文件系统与工作区的关系

```
本地文件系统  =  主系统 / 源文件 / 长期保存
工作区       =  临时处理 / 不长期保存
```

**重要原则**：
- **本地文件系统是唯一权威源**：所有文件最终以本地版本为准
- **工作区只是临时环境**：LLM 工作区里的对话历史和上传文件都是临时的
- **跨 LLM 可迁移**：在新 LLM 中重建工作区时，只需上传本地文件 + 复制项目指令即可完成迁移

## 系统进化工作区

### 定位

维护并持续优化数字分身系统本身，让整个数字分身系统变得更稳定、更准确、更可调用。

**核心任务**：
1. 方案文档和架构的持续优化
2. 系统内核（元认知、元知识、核心人格）迭代
3. 知识库（外部资料档案、skills、笔记）持续整理
4. 配套提示词不断打磨
5. 周期性生成演化迭代报告（月度趋势、季度审查、年度演化）

**不做的事**：不进行具体内容产出。

### 四个核心对话

系统进化工作区内部分四个长期核心对话，各自负责一类工作：

**对话1 — 方案与架构迭代**：调整整体方案、修改文件结构、讨论运行方式。
**对话2 — 配套提示词迭代**：更新项目指令、演化迭代提示词、写作提示词等。
**对话3 — 系统内核迭代**：根据演化迭代结果修订元认知、元知识、核心人格。
**对话4 — 知识库整理**：整理外部资料档案、skills、笔记，筛选元知识候选。

### 迭代节奏

- **月度**：观察变化，生成月度趋势。不直接修改系统内核。
- **季度**：进行人格审查，提出修订建议，实际修改核心人格和元知识。
- **年度**：生成年度演化报告，做版本级修订（核心人格、元认知、元知识、知识库）。
- **触发式**：发现具体问题时随时小修小补。

## 内容产出工作区

### 定位

基于已稳定的数字分身系统，完成具体作品输出。

**核心任务**：文章、小说、资料深度分析等。

**不做的事**：不修改系统本身。

### 调用原则

- **元认知永远优先生效**：任何任务先按元认知规则做调度判断
- **核心人格默认调用**：但根据任务决定哪几个为主、哪几个为辅
- **知识库、元知识、素材记录按需调用**：不每次都全部加载

**典型场景**：
- 时评 → 元认知 + 世界认知/价值排序为主 + 必要素材记录
- 专业分析 → 元认知 + 知识库为主 + 元知识支撑
- 小说 → 元认知 + 兴趣审美/生存哲学为主 + 各类素材按需

## 核心原则总结

- **系统层面**：本地文件系统是主系统，LLM 工作区只是临时环境。任何 LLM 都能运行本系统，只需上传本地文件 + 复制项目指令。
- **架构层面**：四层主架构（系统内核 / 知识创作 / 素材记录 / 演化迭代），加系统说明。层级编号是文件夹排序，不是调用顺序。系统内核扁平化，八文件平行存在。
- **文件维护**：外部资料档案用单文件维护，skills 用文件夹，笔记直接加 md 文件。月总结按"生活记录 / 核心人格 / 外部资料"三块组织。
- **演化治理**：月度趋势只筛选信号，季度审查具有修改建议权，年度报告负责版本判断。小改随时，大改按周期。
- **运行层面**：系统进化工作区决定"我是谁"，内容产出工作区基于此完成"我要说什么"。两者通过本地文件系统流转，不直接互通。
