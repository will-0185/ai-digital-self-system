# FAQ

## Is this a prompt collection?

No. Prompts are only one part of the system. The core is the filesystem architecture and long-term evolution mechanism.

## Is this a chatbot persona?

No. A digital self is not a roleplay persona. It is a structured representation of thinking, judgment, knowledge, creation, and evolution over time.

## How is this different from a memory system?

Memory systems typically store facts and events. This framework goes further — it encodes *how* a person thinks, their meta-cognitive rules, generative knowledge paradigms, and a periodic self-evolution mechanism.

## Do I need a specific AI model?

No. The system is model-agnostic. It can be used with ChatGPT, Claude, Gemini, local models, or future AI systems.

## How do I start?

1. Read `AI_Digital_Self_System.md` to understand the architecture
2. Copy the directory structure to your local filesystem
3. Write your own system kernel (start with meta-cognition rules, then personality)
4. Start recording material monthly
5. Run periodic evolution reviews

## Should I put my private files on GitHub?

Not in a public repository. Use local storage or a private repository for private materials like real monthly summaries, life logs, personal relationships, evidence indexes, and unpublished works.

## Can I use this with my existing notes?

Yes. The material record layer is designed to accommodate existing journals, logs, or notes. The key is to organize them into the three-section format (life / personality / external knowledge) for effective evolution reviews.

## How long does it take to set up?

The initial structure takes minutes. Writing meaningful kernel content (meta-cognition rules, personality dimensions) takes thoughtful effort — typically several hours to days. The system then evolves continuously.
