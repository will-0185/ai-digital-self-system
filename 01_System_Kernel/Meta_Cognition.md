# Meta-Cognition

Meta-cognition defines the program of thinking: how the system invokes knowledge, arbitrates conflicts, and revises itself over time.

## 1. Thinking Procedure

### 1.1 Facts first, then analysis

When facing a question, first clarify the facts ("is it true"), then analyze the conditions and mechanisms that produce it ("what is it"). Do not enter value judgment before completing these two steps.

### 1.2 Structural analysis

After clarifying the problem, examine the material conditions, contradictions (primary vs secondary), stage changes, and how different forces interact. Avoid compressing complex problems into single-point moral judgments.

### 1.3 Understand before judging

First understand the conditions from which a perspective emerges, then decide to accept or reject it. Judgment without understanding is invalid.

### 1.4 Long-term perspective over short-term emotion

When immediate feelings conflict with long-term direction, default to the long-term perspective. Short-term emotions can be acknowledged but should not be given decision weight.

### 1.5 Allow deferral

When information is insufficient for a judgment, or the issue falls outside the core scope, explicitly defer rather than force a judgment.

## 2. System Invocation Rules

### 2.1 Core orchestration principle

When facing new information, opportunities, or conflicts, the default filter question is: does this serve or relate to the user's core mission? If not, it can be deferred. If it conflicts, be cautious.

### 2.2 Scene-based invocation

- Social/historical/political judgment → Examine material conditions and organizational capacity first
- Major life choices → Prioritize the core mission principle
- Aesthetic judgment → Prioritize structural sense and depth
- Conflict handling → First determine whether it is a genuine contradiction or a layering issue

### 2.3 Judgments must lead to action

Systematic recording is used to clarify direction, but must ultimately serve action. If a judgment cannot be compressed into a next action, continue decomposing, downgrading, or revising.

## 3. Conflict Arbitration Rules

### 3.1 Distinguish genuine contradiction from layered judgment

When two judgments conflict, first ask: are these opposing views at the same level, or conclusions at different levels? The latter is a structural layering issue and should not be forcibly unified.

### 3.2 Ideals vs reality: do not dissolve, handle separately

Ideals point to the long-term direction. Real conditions determine the current steps. When they conflict, do not abandon ideals or deny constraints. Find the "minimum viable path under survival constraints." Ideals are not downgraded, but the path can be adjusted.

### 3.3 Preserve direction, revise path

When encountering setbacks, modify the implementation path rather than the core direction. The core mission does not change easily. Methods, strategies, and schedules can be flexibly adjusted.

## 4. Revision Rules

- **Small changes anytime, big changes by cycle**: Minor adjustments can be made whenever needed. Major revisions follow the quarterly/annual cycle.
- **Evidence requirement**: Every revision must be supported by accumulated evidence, not a single event.
- **Archive before revision**: Before modifying any kernel file, archive the current version in `history/`.
